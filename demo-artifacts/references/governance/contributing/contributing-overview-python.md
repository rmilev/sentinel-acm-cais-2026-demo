# LangChain Python — Contributing Overview

*Source: https://docs.langchain.com/oss/python/contributing/overview*

## Introduction

LangChain welcomes contributors of all types. The community invites involvement in bug fixes, feature development, documentation enhancement, and feedback to improve LangChain and LangGraph.

## Main Contribution Pathways

**Reporting Bugs**: Contributors should search existing GitHub issues across LangChain, LangGraph, and Deep Agents repositories before filing new reports. When creating an issue, include a minimal reproducible example and relevant labels. Maintainers will triage and may request additional information.

**Suggesting Features**: Search for existing feature requests first, then start a discussion in the appropriate forum category. Describe the use case, explain the value proposition, and outline relevant test cases.

**Documentation**: Documentation improvements are actively encouraged through a dedicated contribution guide.

**Code Contributions**: Developers can tackle issues labeled "good first issue" or "help wanted." A comprehensive pull request guide is available for first-time contributors.

**Integrations**: Specific guides exist for adding LangChain integrations and Deep Agents sandbox contributions.

## Key Requirements

- Pull requests must link to pre-approved issues or discussions — maintainers will close unlinked PRs
- All submissions require meaningful effort; maintainers will reject low-effort or spam contributions
- Contributors must use the repository's PR template

**Language Policy**: All communication must occur in English to maintain accessibility and searchability across the global community, though perfect grammar isn't required.
